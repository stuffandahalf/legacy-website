Contributing to projects on GitHub

The text with the ' characters are commands to be run using a command prompt.

All commands assume that the command prompt is at the directory of a git repository.

Cloning projects

  1. Open command prompt (cmd.exe or terminal).
  2. Navigate to the directory where you want to clone the project directory.
  3. Execute the command:
    'git clone ${URL}'
    where ${URL} is the url of the github project.

Doing work

  1. Before beginning to do any work, run the command:
    'git pull'
    to get the latest changes to your current branch.
  2. Change the files you want to change.
  3. Once you are ready to push your changes to the repository, Perform the following steps:
    1. Remove any generated files (*.class, *.jar, etc.).
    2. Add your files to the commit by executing:
      'git add .'
    3. Commit your changes by executing:
      'git commit -m "MESSAGE"'
      where MESSAGE is a message describing your changes.
    4. Push your changes:
      'git push'
      following the prompts as they appear.
  4. Verify your files wire pushed by visiting the GitHub repository and checking for your commit.

Working with branches

Making a new Branch

  1. Push or stash any changes you've made in your current branch
  2. Execute:
    'git checkout -b BRANCH'
    where BRANCH is the name of the new branch.
  3. The first time you push to your new branch you will need to execute:
    'git push --set-upstream origin BRANCH'
    where BRANCH is the name of the new branch.

Switching to another branch

  1. Push or stash any changes you made to your current branch.
  2. Run:
    'git checkout BRANCH'
    where BRANCH is the name of the branch you want to switch to

Your local files will be updated to where the branched files are.

Merging your branch

  1. Visit the repository page on GitHub
  2. Switch to the branch you want to merge.
  3. Create a new pull request.
  4. (Optional) Add a comment describing the changes to be merged.
  5. After a period of reviewing the changes, We can safely merge the branches.

If you choose to delete an old branch

  1. Visit the github repository.
  2. Click on the button labeled "n branches" to view all of the branches of the project.
  3. Click on the tab labelled "Yours".
  4. Verify that you won't need the branch anymore,
    as this can only be undone until the page is refreshed.
  5. Press the garbage bin icon.

The branch is now deleted.

Miscellaneous features of Git

Stashing (destroying) your changes

  1. Execute:
    'git stash'

Your files will be reverted to their state at the last commit.

Getting the status of your local files.

  1. Execute:
    'git status'

You will be presented with information about the current branch, and any files that arent known to the repository.

Getting the differences between local and remote files.

  1. Execute:
    'git diff'

The command will present you the changes in files and their names so you know what has changed since the last commit.
